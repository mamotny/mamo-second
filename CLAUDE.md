# CLAUDE.md — プロジェクト方針

このファイルは本プロジェクトでの作業方針を定義する。`/Users/mamorutaniya/.claude/CLAUDE.md` のグローバルルール(言語ポリシー・レスポンス書式・ユーザープロファイル)を継承し、ここで具体化する。

## プロジェクトの目的

- **個人ツール**としてのプロジェクト。mamo 個人の作業を支える用途。
- 中心テーマは **投資先・銘柄分析**(リサーチ・分析が主たる作業)。
- 対象: 個別企業・スタートアップ・株式/ファンドの評価、デューデリジェンス。
- ゴール: 谷家衛(mamo / 投資家)の **投資判断を直接支える** 材料を整備すること。

## 私(Claude)への期待 — 支援スタイル

最優先は **「深掘りリサーチ」**。

- 企業・市場を **多角的に調べ**、出典付きで **網羅的なレポート** を作る。
- 一次情報・複数ソースを当たり、主張には **出典(URL・資料名)** を残す。
- 強気/弱気の両論を提示し、判断の **前提条件** を明記する。
- ただし結論ファースト原則は維持: レポート冒頭に **結論 → リスク → 次のアクション** の要約を置く。
- 定量化を重視する(数値・確率・影響範囲を明示)。「速くなる」ではなく「処理時間が約半分」のように書く。
- 不確実性を隠さない。「なぜ100%でないか」を明示することに価値がある。
- 深掘り依頼時は `deep-research` skill の活用を検討する。

## 分析時の基本姿勢

- ファイナンスのプロ・非エンジニアである mamo 向けに、SWE 専門用語は避け、使う場合は1行で補足する。
- 必要に応じてファイナンスのアナロジーで技術概念を説明する。
- 銘柄・企業を評価する際は、強気/弱気の両論を示し、判断の前提条件を明記する。
- 外部ソースを使う場合は出典を残す。

## 出力フォーマット(厳守)

グローバルの `response-format.md` に従う。ユーザー向け出力は日本語。各トピックを以下のブロックで構成する:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🟩 {短い名詞句のトピックラベル}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
{結論先出しの簡潔な本文}
```

- ヘッダーは結論文ではなく短いラベル(例: `🟩 結論` / `🟩 リスク` / `🟩 次のアクション` / `🟩 想定リターン`)。
- 本文は結論を冒頭1〜2文で述べ、根拠を後置。箇条書きを優先。

## 主用途 — 保有株トラッキング & 朝次レビュー

このプロジェクトの第一の用途は、mamo の **保有株(日本株 / 米国株 / 未上場株)** について
**価格・リサーチ・ニュース** を毎朝まとめてレビューすること。

### データ構造

- 保有銘柄は `holdings/holdings.json`(または `.md`)で管理する。mamo がチャットで伝えた内容を私が起こす。
- 資産タイプ別フィールド:
  - **日本株**: 銘柄名、証券コード(4桁)、保有株数、取得単価(任意)
  - **米国株**: 銘柄名、ティッカー、保有株数、取得単価(任意)
  - **未上場株**: 企業名、保有株数/持分、**評価基準**(直近ラウンド評価額 or 取得簿価)、最終評価日
  - **暗号資産(crypto)**: 名称、ティッカー、保有数量、取得単価(任意)。24時間取引のため終値の概念なし、取得時刻を明記

### 価格・情報の取得方針

- 上場株(日本株・米国株): 株価・基礎指標・ニュースを Web から取得。**価格は15〜20分遅延前提**で、前日終値ベースを基本とする。
- 未上場株: 市場価格なし。値は手動入力の評価基準を据え置き、ニュースは企業名検索で補足。
- 出典(URL・取得時刻)を必ず併記する。

### 成果物フォーマット(両建て)

1. **1枚サマリー**: 全保有の時価・前日比・損益・主要ニュース見出しを一覧化。
2. **銘柄別深掘り**: 気になる銘柄について価格・指標・ニュース・リサーチをまとめた詳細。

毎朝のレビューは「サマリー → 注目銘柄の深掘り」の二段構え。

### 更新頻度

- **毎朝自動**。`schedule` skill / cron で朝の定時にレポートを生成する想定。
- 生成物は日付付きで `reports/YYYY-MM-DD.md` などに保存。
